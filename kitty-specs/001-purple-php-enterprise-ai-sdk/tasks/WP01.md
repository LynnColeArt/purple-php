# WP01: SDK Substrate and Core Contracts

## Objective

Create the foundation for Purple PHP as a Composer-first SDK. This work package should make the repository feel real without prematurely implementing chat, agents, hooks, sidecar, or native runtime.

## Scope

Implement the project skeleton, baseline development tooling, core namespace, test harness, fake provider, and first contract set.

Required contracts:

* `Provider`
* `ProviderRequest`
* `ProviderResponse`
* `SecretStore` or `SecretResolver`
* `PolicyEngine`
* `PolicyDecision`
* `AuditLog`
* `AuditEvent`
* `PromptTemplate`
* `SchemaValidator`
* `SmartFunction`

## Requirements

The repository should include Composer metadata, autoloading, tests, static analysis or linting where practical, and examples directory scaffolding.

The fake provider must be deterministic and suitable for unit tests.

Core interfaces should be narrow and stable enough for WP02 to implement without large rewrites.

## Acceptance Criteria

1. `composer install` succeeds.
2. Test commands succeed.
3. A fake provider can return a deterministic response through the provider interface.
4. Secret values have an explicit redaction behavior.
5. Policy and audit contracts are present and covered by basic tests.
6. No public API requires chat, tool, agent, hook, sidecar, or native runtime classes.
